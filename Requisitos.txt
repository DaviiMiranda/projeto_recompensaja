Este documento serve como o pontapé inicial para o desenvolvimento de uma plataforma full-stack de financiamento coletivo (crowdfunding) baseada em recompensas.

Objetivo Principal: Criar uma aplicação web robusta, segura e amigável que conecte "Criadores" (empreendedores, artistas, etc.) com "Apoiadores" (investidores/doadores). Criadores podem buscar financiamento para seus projetos, e Apoiadores podem financiar esses projetos em troca de recompensas predefinidas.

1. Pilares Tecnológicos

    Backend: Java 21 com Spring Boot

        API: RESTful (Spring Web)

        Segurança: Spring Security (com autenticação baseada em JWT)

        Banco de Dados: Spring Data JPA com MySQL

        Validação: Spring Validation

    Frontend: Angular 

        Estilização: Tailwind CSS (integrado ao Angular)

        Gerenciamento de Estado: Services com RxJS (ou NgRx, se a complexidade exigir)

        Formulários: Angular Reactive Forms

    Banco de Dados: MySQL.

2. Perfis de Usuário (User Roles)

    Apoiador (Backer): O usuário que financia os projetos.

    Criador (Creator): O usuário que cadastra projetos e busca financiamento. (Um usuário pode ser Apoiador e Criador).

    Administrador (Admin): Gerencia a plataforma, aprova projetos e modera usuários.

3. Funcionalidades Essenciais (MVP - Minimum Viable Product)

Abaixo estão as "Epics" ou grandes blocos de funcionalidades.

Módulo 1: Autenticação e Gerenciamento de Usuário (Backend + Frontend)

    User Story (US): Como usuário, quero me cadastrar na plataforma (informando nome, e-mail, senha).

    US: Como usuário, quero fazer login com e-mail e senha.

    US: Como usuário, quero redefinir minha senha caso a esqueça.

    US: Como usuário logado, quero ver e editar meu perfil (foto, biografia, links sociais).

    Backend: Endpoints para /auth/register, /auth/login, /auth/refresh-token. Spring Security deve proteger os endpoints.

    Frontend: Páginas de Login, Cadastro e Perfil. Um AuthGuard para proteger rotas.

Módulo 2: Gerenciamento de Projetos (Fluxo do Criador)

    US: Como Criador, quero um formulário para submeter um novo projeto.

        Campos: Título, Descrição Curta, Descrição Completa (com editor de rich text), Vídeo/Imagem Principal, Meta de Arrecadação (R$), Prazo Limite (data), Categoria (Tecnologia, Arte, Comunidade).

    US: Como Criador, quero definir "Níveis de Recompensa" para meu projeto (ex: R$ 50 - Agradecimento; R$ 100 - Pulseira; R$ 500 - Cupom de 20%).

    US: Como Criador, quero submeter meu projeto para "Aprovação do Administrador" (status inicial: PENDENTE).

    US: Como Criador, quero ter um painel (dashboard) para ver o progresso dos meus projetos ativos (valor arrecadado, nº de apoiadores).

    Backend: API REST para CRUD de Projetos e Recompensas. Endpoints protegidos por role (ROLE_CREATOR).

    Frontend: Formulário reativo de várias etapas para criação de projeto. Página "Meus Projetos" (dashboard do criador).

Módulo 3: Descoberta e Apoio (Fluxo do Apoiador)

    US: Como Visitante/Apoiador, quero ver uma página inicial (Homepage) com projetos em destaque, recentes e mais populares.

    US: Como Visitante/Apoiador, quero uma página de "Explorar" onde posso filtrar projetos por categoria, status (ativos, finalizados) ou buscar por palavra-chave.

    US: Como Visitante/Apoiador, quero clicar em um projeto para ver sua página de detalhes (descrição, meta, valor atual, dias restantes, lista de recompensas).

    US: Como Apoiador, quero selecionar um nível de recompensa, informar o valor e realizar o pagamento (simulação de pagamento no MVP).

    Backend: Endpoints públicos (GET) para listagem e visualização de projetos. Endpoint protegido (POST /projects/{id}/back) para registrar um apoio.

    Frontend: Componentes de Card de Projeto, Página de Detalhes do Projeto, Página de Exploração com filtros.

Módulo 4: Painel de Administração (Fluxo do Admin)

    US: Como Admin, quero um painel para ver todos os projetos com status PENDENTE.

    US: Como Admin, quero aprovar ou rejeitar projetos pendentes (com um campo para feedback em caso de rejeição).

    US: Como Admin, quero poder gerenciar usuários (ex: banir, tornar admin).

    Backend: Endpoints (/admin/**) protegidos com ROLE_ADMIN.

    Frontend: Uma seção separada (ex: /admin) com tabelas e formulários para moderação.

Módulo 5: Funcionalidades Adicionais (Pós-MVP)

    Integração de Pagamento Real: Integração com Stripe, PagSeguro ou Mercado Pago para processar os pagamentos.

    Chatbot: Como descrito, um chatbot (pode ser simples, baseado em regras, ou integrado com uma ferramenta como Dialogflow) para responder FAQs.

    Comentários: Seção de comentários na página de cada projeto.

    Notificações: E-mails transacionais (projeto aprovado, novo apoio recebido, etc.).

4. Modelo de Dados (Entidades JPA Sugeridas)

    User: (id, nome, email, senha, roles[ADMIN, CREATOR, BACKER])

    Project: (id, titulo, descricao, metaValor, valorArrecadado, dataLimite, status[PENDENTE, ATIVO, SUCESSO, FALHOU], criador [ManyToOne com User], categoria [ManyToOne com Category])

    Reward: (id, titulo, descricao, valorMinimo, projeto [ManyToOne com Project])

    Pledge (Apoio): (id, valor, data, apoiador [ManyToOne com User], recompensa [ManyToOne com Reward], projeto [ManyToOne com Project])

    Category: (id, nome)

5. Arquitetura e Estrutura

    Backend (Spring Boot):

        Estrutura: Pacotes por funcionalidade (ex: auth, project, user, payment).

        Padrão: Controller -> Service (lógica de negócios) -> Repository (acesso a dados).

        DTOs: Usar Data Transfer Objects (DTOs) para comunicação entre o Controller e o Frontend, com validações.

    Frontend (Angular):

        Módulos: Estrutura modular (ex: AuthModule, ProjectModule, AdminModule, CoreModule, SharedModule).

        Tailwind: Configurar o tailwind.config.js para o tema do projeto e usar classes utilitárias diretamente no HTML dos componentes.

        Services: Centralizar chamadas de API em services (ex: ProjectService, AuthService).

6. Primeiros Passos (Plano de Ação)

    Setup do Ambiente:

        Criar o projeto Spring Boot (via Spring Initializr) com as dependências (Web, JPA, Security, driver de DB).

        Criar o projeto Angular (via ng new) e adicionar o Tailwind CSS (ng add tailwindcss).

    Desenvolver Módulo 1 (Autenticação):

        Implementar o backend (entidade User, AuthService, endpoints de login/registro, configuração do Spring Security com JWT).

        Implementar o frontend (páginas de login/registro, AuthService no Angular, AuthGuard).

    Desenvolver Módulo 2 (Criação de Projeto):

        Implementar o backend (entidades Project, Reward, Category e seus CRUDs).

        Implementar o frontend (formulário de criação de projeto).

    Iterar: Continuar com os módulos 3 e 4, conectando o frontend ao backend passo a passo.
