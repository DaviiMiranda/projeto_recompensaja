Objetivo: Desenvolver a Single Page Application (SPA) completa para a plataforma "RecompensaJá". O foco é criar uma interface responsiva, moderna e intuitiva, construindo todos os componentes e telas com Angular e estilizando-os com Tailwind CSS.

Stack Tecnológica:

    Framework: Angular 20

    Estilização: Tailwind CSS (utility-first) ja integrado

    Gerenciamento de Estado: Services com RxJS (BehaviorSubject/Observable)

    Formulários: Angular Reactive Forms

    Mocking: Dados simulados em JSON/TS (usando o of() do RxJS nos services).

1. Diretrizes Visuais e Setup (Tailwind)

    Design: O visual deve ser limpo, profissional e inspirar confiança. Priorize legibilidade e fácil navegação.

    Responsividade: O design deve ser "mobile-first", adaptando-se perfeitamente a tablets e desktops.

    Setup:

        projeto ja integrado com a versao tailwind 3.4.17

        Configurar o tailwind.config.js:

            Definir uma paleta de cores primária (ex: primary, secondary) para botões, links e destaques.

            Configurar fontes (ex: Inter, Poppins) para títulos e corpo de texto.

            Usar o plugin @tailwindcss/forms para estilização fácil de formulários.

2. Estrutura de Módulos e Componentes

A aplicação deve ser organizada usando os princípios de módulos do Angular.

Módulo 1: CoreModule

    Objetivo: Conter serviços "singleton" e componentes de layout principais.

    Componentes:

        HeaderComponent: O navbar principal. Deve ser "sticky" e conter:

            Logo.

            Links de navegação (Explorar).

            Barra de busca (simples).

            Lógica condicional:

                Visitante: Botões "Login" e "Cadastre-se".

                Logado: Ícone do perfil com um dropdown (Menu: Meu Painel, Editar Perfil, Sair), mais uma opção de meus projetos na navbar e um para criar projeto.

        FooterComponent: Links institucionais, redes sociais, etc.

    Services:

        AuthService: Deve simular o login/logout (ex: login(email, pass) retorna of(mockUser) após 1 segundo) e armazenar o estado de autenticação (ex: BehaviorSubject<boolean>).
Módulo 2: SharedModule

    Objetivo: Conter componentes, diretivas e pipes reutilizáveis em toda a aplicação.

    Componentes a criar:

        ButtonComponent: Um componente de botão genérico que aceita inputs para diferentes estilos (ex: [style]="'primary'", [style]="'outline'"). (Ótimo para praticar Tailwind @apply ou variantes).

        ProjectCardComponent: O componente visual para listar um projeto. Deve mostrar:

            Imagem principal.

            Título do projeto.

            Nome do Criador.

            Barra de progresso (calculada: valorArrecadado / metaValor).

            Valor arrecadado.

            Dias restantes.

            data de publicação

        InputComponent: Um wrapper para input que se integre com o Reactive Forms e exiba mensagens de erro de validação.
